## The decisions

### Visible failure beats silent failure — but visible-and-actionable is the real standard — *tooling-for-iteration*

All four reviewers independently noted that surfacing the raw failure rather than skipping the day or emitting a blank file was the correct instinct. That instinct is worth codifying: the pipeline should never emit a blank file or silently swallow a failed generation. However, the current failure mode — raw prompt payload with a one-line error banner — is the floor, not the ceiling. A useful failure artifact needs three things the current one lacks: (1) the specific error type from the model response or schema validator, (2) the last successful generation date so continuity can be assessed, and (3) a runnable recovery command. The policy going forward: fail loudly, fail specifically, and give the reader something to run.

### The ADR registry sweep must be verified before generation is attempted — *tooling-for-iteration*

The registry is described as load-bearing — 163 ADRs across all swept repos injected as authoritative ground truth into every generation prompt. The Wednesday failure exposed that there is no checksum, record-count assertion, or completion signal on that sweep before the generation step runs. The context cut off mid-path (`0004-nar` in asset-foundry, ADR-0004), which means either the sweep itself is silently truncating output or the context-assembly step is. Either way, generation was attempted on a malformed input. The required fix: assert that the sweep produced a complete, parseable registry before handing it to the model. A simple record count compared against a known baseline is sufficient to gate the run.

### The daily dev log requires a minimum viable manual override path — *tooling-for-iteration*

Three of four reviewers flagged the same gap: when generation fails, there is no defined fallback. The current options in practice appear to be (a) publish the failure artifact, (b) skip the day, or (c) manually write something without tooling support. Option (a) is what happened Wednesday. Options (b) and (c) have no documented process. The minimum viable recovery path needed: a command that accepts a date, accepts human-written prose as stdin or a file, and produces a valid schema-compliant article without requiring the LLM generation step. This does not need to be sophisticated — a `--manual` flag on the existing generator that bypasses the model and scaffolds from human input is sufficient.

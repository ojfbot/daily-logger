## The decisions

### Wait for a Frame sentinel, not networkidle, before screenshot capture — *tooling-for-iteration*

Playwright's `networkidle` fires when the network has been quiet for 500ms — which can happen before Module Federation remotes have finished mounting their React trees inside the shell host. We replaced the `networkidle` wait with an explicit `waitForSelector` targeting a DOM element that only exists after the Frame shell has fully composed its remotes. This makes the screenshot contract explicit: the image captures the composed state, not the bootstrapping state.

Tags: `visual-regression`, `ci-cd`, `landing`

### Portfolio screenshots must show the composed shell, not standalone apps — *assistant-centric*

jim.software is the first surface a recruiter sees. A screenshot of cv-builder running in its own standalone Vite dev shell looks like a separate app — it erases the Frame OS narrative entirely. Correcting this is a positioning decision as much as a technical one: the screenshot is the primary evidence that Frame is a real multi-app shell, not a collection of unrelated tools.

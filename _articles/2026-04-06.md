## The decisions

### FrameBus (shell #58) is the Phase 1 critical-path item — everything else queues behind it — *assistant-centric*

Shell PR #58 formalises ADR-0013 (cross-domain fan-out and synthesis protocol) and ships a FrameBus proof-of-concept backed by a Playwright e2e suite. FrameBus is a typed postMessage bus: the shell posts a structured event object to sub-app iframes (or Module Federation remotes in the same origin), collects responses via a reply-channel contract, and synthesises the results before surfacing them to the ShellAgent. The 'cross-domain fanout classification fix' corrects a misclassification in the event-routing logic that treated same-origin remotes as cross-origin — the fix matters because the postMessage security model gates on origin; routing same-origin traffic through the cross-origin path adds unnecessary serialisation overhead and, more critically, breaks the synthesis timing assumptions that ADR-0013 encodes. Until this lands, the federation topology is a design document, not running code. The three Phase 1 items that queue behind it — shared component extraction, Storybook, and the header chat bar — all assume a working inter-app event channel. 'Working' means: the shell can publish an event, at least one remote receives it, and a reply reaches the shell within the synthesis timeout. If shared component extraction is attempted before #58 lands, the components exist but nothing can route events to them — the build succeeds locally and the app silently fails to propagate inter-app state. ADR-0013 is readable at `core/adrs/0013-cross-domain-fanout.md`; ADRs 0001–0036 are the full decision record in `core/adrs/`. The ADR explicitly considered BroadcastChannel (rejected: same-origin only, no structured reply channel), a shared Zustand store over Module Federation shared scope (rejected: coupling store shape to the federation topology), and server-sent events (rejected: adds a server round-trip to what should be a client-side coordination primitive). FrameBus via typed postMessage won on the basis that it works across both same-origin and cross-origin boundaries with the same API surface, and the reply-channel contract can be typed end-to-end without a code-gen step.

### tsc --noEmit CI gates in BlogEngine #44 and TripPlanner #43 close the structural gap that let Express 5 failures slip past — *tooling-for-iteration*

Both PRs add a `pnpm type-check` step as a required CI gate before build. The Express 5 param-typing failures in BlogEngine #43 and TripPlanner #42 were caught post-merge precisely because this gate was absent — the build passed while the type-checker would have failed. The pattern to establish: any PR touching `@types/*` or framework packages cannot merge without a green `tsc --noEmit` run. These are 10-minute reviews; neither requires new code. The reason to pair them with shell #58 rather than merge them independently is sequencing hygiene, not a technical dependency — it's cleaner to establish the type-gate fleet-wide in a single pass than to have it live in BlogEngine for a week before TripPlanner catches up.

### MrPlug #36 (CVE remediation) must merge before #37 (CI trigger change) — order is a security invariant — *security-as-emergent-ux*

MrPlug #36 patches three CVE-class vulnerabilities: a critical RCE via happy-dom (bumped 15→20.8.9), two high-severity path-traversal issues via rollup and `@crxjs/vite-plugin`. MrPlug #37 switches the Claude Code review workflow from `pull_request` to `workflow_dispatch`. The ordering matters because `workflow_dispatch` means the workflow runs only on manual trigger — but the window between merging #37 and merging #36 is a window where anyone with repo write access (including an automated process) could manually trigger the new workflow against the unpatched dependency tree. The attack surface is narrow but real: a triggered run that invokes happy-dom against untrusted input during that window would have the RCE exposure present. Merge #36 first, confirm `pnpm audit --audit-level=high` reports zero findings, then merge #37. Reversing the order for a 10-minute gap is not worth the exposure.

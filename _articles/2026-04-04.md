## The decisions

### Patch @xmldom/xmldom via npm overrides, not a direct version bump — and verify before calling it closed — *security-as-emergent-ux*

GHSA-wh4c-j3r5-mjhp is a vulnerability in a transitive dependency — xmldom isn't in cv-builder's direct dependency list, so bumping a package.json entry isn't the right fix. Adding an overrides field forces npm's resolution algorithm to satisfy the constraint for the entire subtree. The alternative — waiting for an upstream package to release a patched version — is not acceptable when a CVE is live and the repo is publicly visible. On the exposure question: cv-builder uses xmldom at build time to process developer-supplied input, not at runtime against untrusted user input. That lowers the blast radius, but it does not lower the fix priority — the repo is public and the dependency is auditable by anyone. The fix was verified on a clean install before this entry was written: npm audit --audit-level=high returned zero high-severity findings after the pin.

Tags: `hardening`, `cv-builder`

### Separate the security pin commit from the type-fix commit — *security-as-emergent-ux*

Commits 27fc4e4 (the overrides pin) and 00a0a14 (the explicit xmldom type import) are intentionally separate. The security fix is a one-line change to package.json that needs zero review friction — it must be mergeable by itself. The type fix is a TypeScript compatibility patch that requires understanding the import path change. Mixing them into one commit would either delay the security patch or push an under-reviewed type change through on its coat-tails. This is the discipline: a security patch and a compatibility patch have different audiences and different urgency. They should never share a commit.

Tags: `hardening`, `cv-builder`

### Force dark colour scheme in Playwright screenshot capture — *tooling-for-iteration*

The landing page screenshot pipeline captures app previews via Playwright and commits them as static assets. Several apps (daily-logger, SEH Study) respect prefers-color-scheme — without colorScheme: 'dark' set on the browser context, Playwright defaults to light mode and screenshots look wrong against the dark background. This is a single-line config change with outsized visual impact: every future capture is now consistent without any manual override per-app. The tag 'module-federation' on the earlier draft was a mislabel — this has nothing to do with Module Federation. The correct tags are landing and tooling. The pipeline is still a manual tool; automation remains an open action from 2026-03-31.

Tags: `landing`, `tooling`

### BioCard as a separate component between hero and portfolio, not inline bio text — *assistant-centric*

The bio content needed to expand without reflowing the hero section or the portfolio grid. Extracting it as a standalone BioCard component with its own expand/collapse state means the portfolio grid layout is unaffected by bio length, and the expand animation (300ms ease-out CSS height transition + scroll-into-view) is encapsulated. Inline text would either truncate or push the grid below the fold on first load. No motion spec exists — the easing and duration were set from intuition and confirmed manually. If Frame adopts a formal motion system, BioCard will need to be revisited.

### READMEs as first-class design artifacts — lead with the problem, not the implementation — *model-behavior-as-design*

Across five repos (MrPlug, purefoy, GroupThink, core, daily-logger), the pattern is identical: the old READMEs led with what the thing is, and the new ones lead with why it exists and what problem it solves. This is the same discipline that governs agent prompt design — the first token sets the frame. A recruiter landing on MrPlug from a portfolio link has 10 seconds; if the README opens with 'a Chrome extension for…' instead of 'closes the feedback loop between design review and code change,' the context is lost. GroupThink is the benchmark README from this sweep: it has a live screenshot, a concrete capability claim (64 tabs, semantic grouping), and a getting-started path that works without prior context. The canonical ecosystem table in CLAUDE.md — the structural source that would prevent the next sweep from drifting — was not created Friday. The decision to defer: it requires agreeing on schema (repo name, port, one-line description, phase, status) before writing a line. That's a 20-minute design decision that would have broken the sprint's momentum. It is deferred explicitly, not forgotten.

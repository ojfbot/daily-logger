## The decisions

### session-init.sh symlink tracking is now fleet-complete — *tooling-for-iteration*

Every active Frame repo — lean-canvas, TripPlanner, BlogEngine, MrPlug, core-reader, seh-study, purefoy — merged the symlink-tracking PR this week. Mechanically: `core/scripts/install-agents.sh` section 6 creates the symlink, and the symlink itself is committed as a tracked git object in the repo's index. When git checks out the branch or a developer does a fresh clone, git restores the symlink file from the index — no post-checkout hook or `.gitattributes` rule required. Without the tracked symlink, a developer cloning a repo for the first time silently loses the session-init hook and gets no onboarding scaffold at session start — a failure mode that produces no error and is hard to diagnose because git simply skips untracked hook files.

Honest acknowledgment: the article previously noted this fleet rollout demonstrates exactly the maintenance burden that `@ojfbot/core-skills` (#77) as an npm package would eliminate. The symlinks shipped first because iteration speed mattered more than distribution cleanliness at the time — the npm package migration is the correct follow-on, and this week's rollout makes that urgency concrete.

### purefoy #39: ProgressReporter does not block the transcription hot path — *model-behavior-as-design*

The GPU transcription observability layer writes `progress.json` to a sidecar file at each of 8 pipeline stages rather than blocking the main transcription loop or adding in-process latency. This is the correct pattern for long-running ML pipelines: decouple the observation plane from the execution plane so a slow filesystem write cannot add latency to GPU-bound work. The alternative — inline progress callbacks that block on acknowledgement — would serialize transcription segments with disk I/O and degrade throughput on multi-hour podcast runs.

What this unblocks for Phase 7: the `progress.json` output at 8 stages is the data contract a Frame UI needs to render transcription progress to a user. The `preflight()` check means the agent won't silently consume GPU time on a misconfigured environment. The remaining Phase 7 gap is the Frame integration itself: purefoy is not yet a Module Federation remote. This is a parallel track to the observability work, not a downstream dependency — purefoy's `remoteEntry.js` and webpack remote config have not yet been stubbed. Whether this is an afternoon of config or a structural design decision about how purefoy exposes its progress stream to Frame has not been fully scoped; that scoping should happen before the next purefoy sprint.

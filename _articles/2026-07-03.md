## The decisions

### Generation pipeline must validate output before treating it as publishable — *tooling-for-iteration*

The current pipeline has no gate between model output and publication. Thursday's failure mode — emitting raw prompt context instead of structured article data — made it through to the council review queue. The correct architecture is: schema validation immediately after generation, with an explicit error branch that produces a stub (title, failure reason, raw commit summary) rather than swallowing the error silently. 'Invalid data' is a symptom, not a diagnosis; the error branch must capture where in the pipeline validation failed.

### ADR registry injection strategy needs a retrieval filter, not a full-corpus dump — *tooling-for-iteration*

Injecting all 162 ADR entries raw into every generation prompt is the most likely proximate cause of Thursday's failure — context pressure from an oversized registry increases the probability of malformed output. The correct fix is a retrieval step that filters the registry to ADRs relevant to the day's active repos and commits before injection. The full registry remains authoritative ground truth; only the slice relevant to today's work enters the prompt. This is a pipeline architecture change, not an ADR content change.

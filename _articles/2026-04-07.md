## The decisions

### FrameBus ships as a bundle: ADR-0013 + implementation + Playwright e2e in one PR — *tooling-for-iteration*

Shell PR #58 ships the FrameBus PoC, the cross-domain fanout spec, and the Playwright e2e suite together — not as three PRs across three weeks. The rejected alternative was the standard pattern: merge the PoC first, write the ADR 'when there's time,' add tests in a follow-up. That pattern produces PoCs that get copy-pasted into production before their constraints are documented and tested. Bundling enforces a contract: the decision record must be coherent enough to generate a test before the code ships. The Playwright suite runs against a localhost dev server in CI — smoke-test confidence, not full integration, but sufficient to validate the message contract and serve as a merge gate.

Tags: `module-federation`, `cross-app-orchestration`

### gcgcca adopts Pydantic → OpenAPI → TypeScript as the canonical cross-language type bridge — *model-behavior-as-design*

Rather than hand-writing TypeScript types for USGS API responses, scripts/generate_schema.py exports 16 Pydantic v2 models to openapi.json (using pydantic's native schema export — no running server required) and then to TypeScript. Single source of truth: change a Pydantic model, run the script, TypeScript types update; anything that relied on the old shape fails at compile time, not at 2am in production. The failure mode to watch: Pydantic models that use custom validators or Union types with discriminators may not map cleanly to OpenAPI — those cases require manual schema annotation. This pattern follows the purefoy CLI+UI reference implementation and should be the template for any future Python-backed Frame sub-app. An ADR documenting this pattern, its constraints, and its failure modes is a suggested next action.

Tags: `type-bridge`, `module-federation`

### Skill ## See Also sections create a distributed activation graph, not a central dispatch table — *tooling-for-iteration*

Adding ## See Also postflight sections to 18 isolated skills means each skill's output text contains the trigger phrases for the next skill in the workflow. Concrete example: /resume-audit outputs 'run /skill-gap to cross-reference against a target role' — Claude Code reads that phrase and surfaces /skill-gap without the developer remembering the sequence. The alternative (a central dispatch table) requires updating one file every time a new skill is added. The distributed approach gives each skill ownership of its outgoing edges. The observable signal this works: in practice, skill suggestion latency dropped to zero — the next skill appears in the same response turn. Formal telemetry (tracking which ## See Also suggestions were accepted) is not yet instrumented.

Tags: `skill-flywheel`, `model-behavior-as-design`

### @frame/tsconfig and @frame/eslint-plugin as cross-repo shared presets — *tooling-for-iteration*

cv-builder publishes @frame/tsconfig shared presets (browser, node, node-emit) and @frame/eslint-plugin with 5 custom rules. Shell PR #59 immediately migrated four package tsconfigs to extend these presets, and CI passed clean. The tradeoff is explicit: any breaking change to the shared preset now propagates to every consumer simultaneously — but drift across 13 independent tsconfig.json files is the worse failure mode, especially when an interviewer is reading your CI logs. The /validate action below is the forward guard: run tsc --noEmit across all shell packages before adding more consumers, to catch regressions before they compound.

Tags: `tooling-for-iteration`

### Claude Code review switched to workflow_dispatch in MrPlug — *security-as-emergent-ux*

Changing from pull_request to workflow_dispatch means Claude Code review fires only on explicit invocation. This is a cost-control decision: the pull_request trigger was running Claude Code review on every lockfile bump and bot PR, burning API quota with no return. The design principle this encodes: automated AI actions should have explicit invocation surfaces. The cost of the Monday sprint in CI API calls is not currently instrumented — that's a gap. The workflow_dispatch change is the first cost-control gate in the fleet.

Tags: `security-as-emergent-ux`

### ADR-0036 codifies lock-file-rebuild as a two-gate CI requirement — *security-as-emergent-ux*

Every lock file rebuild must pass both tsc --noEmit and pnpm audit --audit-level=high before merge. Motivation: Express 5 type-breaking changes were caught post-merge last week because the type-check gate didn't exist. The tradeoff of requiring two gates instead of one: rebuild PRs take longer and occasionally surface pre-existing type errors that aren't related to the lockfile change — those become blocking noise. The decision accepts that tradeoff because the alternative (a type-breaking dependency change that merges silently) is harder to debug and more expensive to roll back than fixing pre-existing errors at PR time.

Tags: `security-as-emergent-ux`, `tooling-for-iteration`

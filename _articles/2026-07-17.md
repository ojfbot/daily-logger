## The decisions

### Correct the Node floor rather than loosen the ESM constraint — *tooling-for-iteration*

The `ERR_REQUIRE_ESM` error in vitest's jsdom environment is a Node version problem disguised as an import error. Recent jsdom versions use dual CJS/ESM packaging; on Node versions below the corrected floor, the module resolver picks incompatible paths for different internals, producing the error at vitest startup. Three alternatives were considered and rejected: (1) `--experimental-vm-modules` — masks the underlying engine requirement rather than surfacing it, and experimental flags are unstable across Node versions, creating a different class of future breakage; (2) pinning an older jsdom version — trades the correct fix for a dependency freeze that accumulates its own debt; (3) doing nothing — leaves the quality gate invisible. Raising `engines.node` in `packages/renderer/package.json` is the honest fix: it documents the actual requirement, prevents the same failure on any future CI agent or contributor shell running a lower Node version, and costs nothing to revert if the floor needs to move again.

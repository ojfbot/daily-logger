Fifty-eight commits Monday — one of the highest-volume days in the project's history — and nearly all of it converges on the same theme: the shared component layer is real now, and the fleet is adopting it. `SidebarToggle` from `@ojfbot/frame-ui-components` replaced inline implementations in every sub-app simultaneously. The shell gained deep-link routing so landing-page cards can open specific apps directly. And daily-logger shipped an editorial revision sidebar with its own CI workflow — this blog is now editing itself.

## The decisions

### Fleet-wide SidebarToggle adoption in a single sprint — *tooling-for-iteration*

Rather than letting shared-component adoption happen organically repo by repo, we drove `SidebarToggle` across all seven sub-apps in one coordinated sprint. Mechanically, adoption means each sub-app updates its import to consume `SidebarToggle` from `@ojfbot/frame-ui-components`, which is served at runtime via Module Federation from the shell — not a version-pinned package install. This distinction matters: if the shell redeploys a breaking change to `frame-ui-components`, sub-apps that haven't redeployed will consume the new version on next load. The coordinated sprint eliminates the divergence window where some sub-apps use the shared component and others carry inline copies, but it does not eliminate the runtime coupling risk. That risk is the open design question for the shared layer as it grows.

Tags: `shared-component-adoption`, `phase-1`, `frame-ui-components`, `module-federation`

### CLAUDE.md deployment guardrail as security-emergent UX — *security-as-emergent-ux*

Every repo now carries a `CLAUDE.md` section that explicitly prohibits direct CLI production deploys (`vercel deploy --prod`, `vercel promote`). The guardrail is written in the system prompt surface the AI agent reads — not just a README note a human skims. This means the agent is instructed not to trigger an unsafe production operation, which is exactly what security-as-emergent-UX means: the constraint is built into the interaction layer, not bolted on after. An honest caveat: `CLAUDE.md` is a prompt instruction, not a hard technical control. Nothing in the environment prevents the deploy if the CLI is available and authenticated, and a Claude session running without the file loaded would have no constraint at all. The CLAUDE.md guardrail is a process control. The structural backstop — a CI gate that would catch and block a rogue prod deploy — does not yet exist. TripPlanner #16 (IDOR auth check) and #19 (security scan patterns) are the open structural security work; the CLAUDE.md guardrails and those issues address different layers and neither substitutes for the other.

### Removing CSS transform from frame-fade-in to fix position:fixed stacking context — *tooling-for-iteration*

The `.frame-fade-in` animation used `transform: translateY(8px)` which created a new CSS stacking context, breaking `position:fixed` elements inside sub-apps — overlays, modals, and sticky headers would scroll with content instead of the viewport. Switching to opacity-only animation removes the stacking context entirely. The fix is in the shell's animation stylesheet (the `.frame-fade-in` class definition); the symptom appears only in sub-apps, making it very hard to diagnose without knowing the CSS transform/stacking-context rule. Debugging tip: if `position:fixed` elements in any sub-app are scrolling with content rather than the viewport, check whether any ancestor in the shell applies a CSS `transform`, `filter`, `perspective`, or `will-change: transform` — any of these promotes the element to a new stacking context and invalidates fixed positioning for all descendants.

### purefoy ADR-010: S3 + CloudFront for transcript serving — *tooling-for-iteration*

At 348 episodes and 96 MB, purefoy's transcript corpus is too large to bundle or serve from the Express API on every request. ADR-010 externalises it to S3 + CloudFront: an export script in the purefoy repo produces S3-ready structured JSON (run details not yet documented outside the PR description), the API returns a `PaginatedResponse` envelope with camelCase keys, and CloudFront handles CDN caching. The alternative — embedding transcripts in the bundle — would bloat every cold start and prevent the eventual podcast AI agent from streaming transcript chunks on demand. Season groupings were broken due to malformed/absent grouping data, not a logic error; profession grouping is now the primary browse pattern. ADR-010 is referenced by number throughout the codebase but the versioned ADR document does not yet exist in the purefoy repo — it lives in the PR description. Cost at current traffic and at projected AI-agent request volume is not yet documented; this is a known gap before the streaming agent work begins.

## The decisions

### Vault lint promoted from advisory to a commit gate (ADR-0089) — *model-behavior-as-design*

The vault lint check now runs in --gate mode on the push path in selfco-box, exiting 1 on two classes of violation: broken [[links]] (a wikilink whose target note does not exist in the vault) and raw-without-source violations (a note in the raw/ ingestion zone that lacks a source:: frontmatter field — the rule is a frontmatter key check encoded in vault_lint.py). Without a gate, vault integrity degrades silently: the staleness scanner (ADR-0080) and semantic suggester (ADR-0091) both read wiki pages as authoritative, so a broken link or unsourced raw note propagates as a confident answer into future cultivate passes. Treating the commit gate as a behavior contract — not just a CI check — is what the model-behavior-as-design tag marks: the lint rule defines what the system is allowed to believe.

### Defuddle ingest as shadow mode, not a replacement (ADR-0090) — *tooling-for-iteration*

The motivation for reaching for defuddle at all: raw URL captures were producing noisy content for the embeddings channel — boilerplate, nav chrome, cookie banners — that degraded semantic-suggest candidate quality on URL-sourced notes. Defuddle's readability extraction strips that noise. The reason it ships as shadow mode rather than immediate replacement: the raw ingest path carries frontmatter, inline tags, and vault conventions that downstream tools (lint gate, cultivate, staleness scanner) depend on. Replacing it immediately would silently break those consumers. Shadow mode logs the readability-extracted content alongside the raw capture — currently to a structured log file in the selfco-box run directory — so the team can evaluate extraction quality over real captures before any promotion decision is made. The /validate suggested action specifies 5 real ingests as the minimum sample; the promotion criterion is that extraction quality is visibly better than raw on noisy URL sources without dropping structured metadata. That criterion is intentionally qualitative at this stage because the vault is small enough that a precision number would be noise.

### Semantic link-suggester feeds cultivate as optional input, not a required step (ADR-0091) — *assistant-centric*

The semantic-suggest.py embeddings channel surfaces cross-cluster candidates to the cultivate pass but does not block or replace it — the daily cultivate run succeeds whether or not the suggester produces output. The failure mode of making it required on a small vault is concrete: embedding quality is proportional to vault density. On a sparse vault, the embeddings cluster tightly around recency rather than semantic distance, and the top-N candidates are mostly recent notes that are already linked. A required step that produces only noise on day one would train the user to ignore cultivate output entirely, destroying the value of the whole pass. Making it optional means cultivate is useful from day one and the suggester's contribution can be measured independently. The baseline measurement ADR-0091 needs: run semantic-suggest.py against the live vault, count how many suggested cross-cluster links survive manual review as genuinely non-obvious, and record the precision rate. If precision is below ~30%, the threshold or embedding model needs tuning before the suggester is worth promoting. If it clears ~60%, it's a candidate for opt-in default.

### daily-logger collect step: preflight via rate_limit, fail loud on token rejection — *tooling-for-iteration*

The collect step now calls the rate_limit endpoint before the main sweep so the Actions installation token is validated early, and any token rejection is now a hard failure rather than a silent skip. The previous behaviour was a bare exception swallow on the token-check path: a rejected token raised an auth error, the except block logged nothing and continued, and the sweep ran against zero repos. The root cause of the 2026-05-19→06-09 outage window — 22 days, zero articles produced, no visible CI failure — was this single swallow. The fix is two changes: add a rate_limit preflight call that will fail the job visibly if the token is bad, and convert the swallow to a hard sys.exit(1) with an explicit error message. The backfill PR (#209) is open alongside the sweep registration fix (#210). #209 is independently mergeable — the backfill reconstructs from git history and is not blocked on #210.

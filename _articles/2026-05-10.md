## The decisions

### ADR-0012: ResearchCurator as an upstream brief-generation sub-agent — *model-behavior-as-design*

asset-foundry's previous agent roster operated entirely in the execution layer — generation, validation, sync. ADR-0012 proposes a new upstream sub-agent whose job is to translate visual intent (reference images, production notes, stylistic constraints) into a structured design brief before any geometry is generated. The rationale is that prompt quality is the dominant variable in asset fidelity, and a dedicated brief-generation step prevents that work from being done ad-hoc in the chat window.

ADR-0012 is currently Proposed. Its disposition is actively undecided — not parked, but not assigned to anyone for acceptance review, and no acceptance criteria are written down yet. That is the gap the /adr action item below is meant to close: either name the acceptance criteria and assign review, or formally park it with a stated reason. Until one of those happens, it is functionally stale despite being recently filed.

The addition of an upstream agent also has sequencing implications that the article should name plainly: introducing ResearchCurator before asset-foundry's Frame MF front door is visible in the dev environment means there is no way to validate the brief-to-generation pipeline end-to-end. The port collision fix is a prerequisite for meaningful ResearchCurator prototyping, not just for Frame MF visibility.

### Proposed status is load-bearing — not a placeholder — *tooling-for-iteration*

ADR-0012 is filed as Proposed rather than Accepted, which in this project's ADR convention means the decision is under active consideration but has not passed implementation review. ADRs live in the repo's `docs/adr/` directory and advance through Proposed → Accepted via an explicit implementation review — not by default over time. The convention exists because prior sessions began writing code against Proposed ADRs, treating the proposal as a working implementation spec. When the ADR shifted during review, that code became drift. The Proposed/Accepted gate is the mechanical check against that pattern.

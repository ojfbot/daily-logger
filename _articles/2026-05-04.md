Two Module Federation remotes claiming the same port means one silently fails to load at runtime. In Webpack Module Federation, the host resolves remotes by the URL in remoteBase at build time — there is no runtime tie-breaking. The remote whose URL is stale or wrong simply fails to fetch its remoteEntry.js; the error surfaces in the browser console as a network failure on the remoteEntry request, not as an MF-specific error.

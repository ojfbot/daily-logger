The Notion inbox write channel is a deliberate architectural choice, not a placeholder. Rather than writing directly to the Obsidian vault filesystem from a Claude session, all chat-originated captures route through the Notion `📥 selfco — Inbox` page. The selfco-box daemon — a process running on a local machine, not a cloud function or container — then ingests from Notion into the vault via a path-jailed Claude agent. "Path-jailed" here means the agent's tool configuration carries an explicit allowlist restricting filesystem writes to the vault directory boundary; writes outside that boundary produce a tool-permission error, not silent failure. One honest gap to name: Notion is a paid SaaS product with API rate limits, and the selfco-box daemon makes programmatic calls against it. The current deployment is single-user and well within free-tier limits, but the Notion dependency is a real cost surface if this pattern scales. That tradeoff was accepted knowingly — Notion was chosen over a local queue file or git-committed scratch pad because it provides a human-readable, mobile-accessible inbox that works across iOS and desktop before the daemon runs. The API dependency cost is the price of that access pattern.

## The decisions

### Notion Inbox as the canonical chat-to-vault write channel — *security-as-emergent-ux*

Rather than writing directly to the Obsidian vault filesystem from a Claude session, the design routes all chat-originated captures through a Notion `📥 selfco — Inbox` page. The selfco-box daemon then ingests from Notion into the vault via a path-jailed Claude agent — where the jail is an explicit tool-configuration allowlist restricting writes to the vault directory boundary. Writes outside that boundary produce a tool-permission error, not silent failure. This keeps the write path auditable at the Notion layer and the vault filesystem clean from direct LLM writes.

The tradeoff accepted: Notion is a third-party SaaS dependency in the write path. If Notion is unavailable, captures are not queued locally — they are dropped at the session layer. That failure mode is known and accepted at current single-user scale. The choice of Notion over simpler intermediaries (local queue file, git-committed scratch pad, direct MCP write with audit log) was made for human-readable, mobile-accessible inbox access across iOS and desktop before the daemon runs. The Notion dependency is currently within free-tier API limits; it becomes a real cost surface if this pattern scales to multiple users or vaults.
